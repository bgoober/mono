GitHub Repository Analyzer

DevPie analyzes the contributions and commits to a GitHub repository.

The output is meant to be used as the basis of a suggested revenue split amongst the project's contributors.

Points are awarded to authors and committers based on lines added and deleted:

  • 5 points for boilerplate commits, 100 points for non-boilerplate
  • +25 points for each line added (non-boilerplate commits only)
  • +50 points for each line deleted (non-boilerplate commits only)
  • +25 points for verified commits

Boilerplate commits are identified by specific keywords in the commit message and should be committed separately with no changes from its initial state. GitHub, automated workflows, and bots are excluded from the Pie.

Given the financial incentives and importance of diligent participation in the project's development, it is recommended to implement strict branch protection and commit rules to maintain the integrity of your project and each contributor's share.
